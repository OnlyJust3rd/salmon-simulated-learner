A relational database organizes data into tables made of rows and columns. Each
table represents an entity and each row is a tuple describing a single record.
Columns define attributes and every attribute draws its values from a domain. A
schema specifies the structure of the tables and the relationships between them.
A primary key uniquely identifies each row within a table.
